# 📝 Jira Interview Questions & Answers (DevOps)

This section contains commonly asked **Jira interview questions for DevOps Engineers (Fresher – 2+ Years Experience).**

---

# 1. What is Jira?

Jira is a **project management and issue tracking tool developed by Atlassian**.

It is widely used by development and DevOps teams to:

- Track bugs
- Manage tasks
- Plan sprints
- Monitor project progress

---

# 2. What is Jira mainly used for?

Jira is used for:

- Issue tracking
- Bug tracking
- Agile project management
- Sprint planning
- Task management

---

# 3. What is an Issue in Jira?

An issue represents a **task, bug, or work item that needs to be completed**.

Examples:

- Bug
- Task
- Story
- Epic

---

# 4. What are Issue Types in Jira?

Common issue types:

- Bug
- Task
- Story
- Epic
- Sub-task

---

# 5. What is an Epic?

An Epic is a **large body of work that can be broken down into smaller tasks or stories**.

Example:

```
Epic → User Authentication System
```

---

# 6. What is a Story in Jira?

A story represents **a feature or functionality from the user's perspective**.

Example:

```
User Story → User should be able to log in using email and password
```

---

# 7. What is a Task in Jira?

A task is a **specific piece of work that needs to be completed**.

Example:

```
Setup CI/CD pipeline
```

---

# 8. What is a Bug in Jira?

A bug is an **error or defect in the application that needs to be fixed**.

Example:

```
Login page not working
```

---

# 9. What is a Sprint?

A sprint is a **fixed time period (usually 2 weeks) during which a team completes planned work**.

It is used in Agile development.

---

# 10. What is a Jira Board?

A board is used to **visualize work and track progress of tasks**.

Types of boards:

- Scrum Board
- Kanban Board

---

# 11. What is a Scrum Board?

A Scrum board is used in **Agile Scrum methodology to manage sprint tasks**.

Columns usually include:

```
To Do → In Progress → Done
```

---

# 12. What is a Kanban Board?

A Kanban board is used to **visualize workflow and limit work in progress**.

It focuses on continuous delivery.

---

# 13. What is Workflow in Jira?

A workflow defines **the lifecycle of an issue from creation to completion**.

Example workflow:

```
To Do → In Progress → Review → Done
```

---

# 14. What is Jira Backlog?

The backlog is a **list of tasks or issues that need to be completed in the future**.

Teams select items from the backlog for upcoming sprints.

---

# 15. What are Jira Labels?

Labels are **tags used to categorize and filter issues**.

Example:

```
devops
bug
backend
```

---

# 16. What is a Jira Ticket?

A Jira ticket is another name for **an issue or task tracked in Jira**.

Example:

```
DEVOPS-101
```

---

# 17. What is Jira Dashboard?

A dashboard provides **visual reports and metrics about project progress**.

Examples:

- Sprint progress
- Issue statistics
- Workload distribution

---

# 18. Real DevOps Scenario

### Problem

Production deployment failed.

### Solution

1. Create a Jira ticket
2. Assign issue to DevOps engineer
3. Investigate deployment logs
4. Fix pipeline or infrastructure
5. Update Jira ticket status

---

# 19. Benefits of Jira

- Better project tracking
- Improved team collaboration
- Agile project management
- Issue tracking
- Reporting and dashboards

---

Jira helps DevOps teams:

- Track deployment issues
- Manage tasks and bugs
- Coordinate work between teams
- Monitor sprint progress
