# 📊 Prometheus Interview Questions & Answers (DevOps)

This section contains commonly asked **Prometheus interview questions for DevOps Engineers (Fresher – 2+ Years Experience).**

---

# 1. What is Prometheus?

Prometheus is an **open-source monitoring and alerting system** used to collect and store metrics from applications, servers, and services.

It stores metrics as **time-series data** and allows querying using **PromQL (Prometheus Query Language).**

---

# 2. What are Metrics?

Metrics are **numerical measurements collected over time**.

Examples:

- CPU usage
- Memory usage
- Disk usage
- Network traffic
- Request count

---

# 3. What is Time-Series Data?

Time-series data is **data points indexed by time**.

Example:

```
cpu_usage{instance="server1"} 75
```

This shows CPU usage for server1 at a specific time.

---

# 4. Prometheus Architecture

Main components:

- Prometheus Server
- Exporters
- Alertmanager
- Pushgateway
- Visualization tools (Grafana)

---

# 5. What is Prometheus Server?

Prometheus Server is responsible for:

- Scraping metrics from targets
- Storing metrics
- Running queries
- Generating alerts

---

# 6. What are Exporters?

Exporters are **tools that expose metrics for Prometheus to collect**.

Examples:

- Node Exporter
- MySQL Exporter
- Blackbox Exporter
- Kubernetes Exporter

---

# 7. What is Node Exporter?

Node Exporter collects **system-level metrics from Linux servers**.

Examples of metrics:

- CPU usage
- Memory usage
- Disk usage
- Network statistics

Default port:

```
9100
```

---

# 8. What is Prometheus Scraping?

Scraping is the process where **Prometheus periodically collects metrics from targets via HTTP endpoints.**

Example endpoint:

```
http://server-ip:9100/metrics
```

---

# 9. What is PromQL?

PromQL (Prometheus Query Language) is used to **query and analyze metrics stored in Prometheus.**

Example query:

```
rate(http_requests_total[5m])
```

This calculates the rate of HTTP requests over the last 5 minutes.

---

# 10. What is Alertmanager?

Alertmanager handles alerts generated by Prometheus.

It sends alerts to systems like:

- Email
- Slack
- PagerDuty
- Webhooks

---

# 11. What is Pushgateway?

Pushgateway allows **short-lived jobs to push metrics to Prometheus**.

Useful for batch jobs.

---

# 12. What is Service Discovery in Prometheus?

Service discovery automatically **detects monitoring targets dynamically**.

Supported platforms:

- Kubernetes
- AWS EC2
- Consul
- DNS

---

# 13. What is a Prometheus Target?

A target is an **endpoint from which Prometheus collects metrics**.

Example:

```
http://server-ip:9100/metrics
```

---

# 14. Prometheus Default Port

Prometheus runs by default on:

```
9090
```

Example access:

```
http://server-ip:9090
```

---

# 15. Example Prometheus Configuration

Example `prometheus.yml` file:

```yaml
global:
  scrape_interval: 15s

scrape_configs:
  - job_name: "node"
    static_configs:
      - targets: ["localhost:9100"]
```

---

# 16. How Prometheus Works

Flow:

```
Application / Server
        ↓
    Exporter
        ↓
   Prometheus Server
        ↓
      Grafana
```

---

# 17. Prometheus Integration with Kubernetes

Prometheus can monitor:

- Nodes
- Pods
- Containers
- Services

Tools used:

- kube-state-metrics
- node-exporter
- cAdvisor

---

### Problem

Application response time is increasing.

### Solution

1. Check Prometheus metrics
2. Analyze request rate and latency
3. View Grafana dashboard
4. Identify bottlenecks
