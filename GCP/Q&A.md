# ☁ Google Cloud Platform (GCP) Interview Questions & Answers (DevOps)

This section contains commonly asked **GCP interview questions** for DevOps engineers.

---

# 1. What is Google Cloud Platform (GCP)?

Google Cloud Platform (GCP) is a **cloud computing platform provided by Google that offers services such as computing, storage, networking, and machine learning**.

It allows users to build, deploy, and scale applications globally.

---

# 2. What are the main services provided by GCP?

GCP provides various services such as:

- Compute Engine
- Google Kubernetes Engine (GKE)
- Cloud Storage
- Cloud Functions
- Cloud SQL
- VPC
- Cloud Monitoring

---

# 3. What is Compute Engine?

Compute Engine is a **virtual machine service in GCP used to run applications on scalable infrastructure**.

It is similar to AWS EC2.

---

# 4. What is Google Kubernetes Engine (GKE)?

GKE is a **managed Kubernetes service in GCP used to deploy and manage containerized applications**.

Google manages the Kubernetes control plane.

---

# 5. What is Google Cloud Storage?

Google Cloud Storage is an **object storage service used to store and retrieve data**.

It is similar to AWS S3.

---

# 6. What is Cloud Functions?

Cloud Functions is a **serverless compute service that runs code in response to events**.

It automatically scales based on demand.

---

# 7. What is Cloud SQL?

Cloud SQL is a **managed relational database service** that supports:

- MySQL
- PostgreSQL
- SQL Server

---

# 8. What is a GCP Project?

A project is a **logical container for resources in GCP**.

All resources such as VMs, storage, and services are organized within a project.

---

# 9. What is Identity and Access Management (IAM) in GCP?

GCP IAM is used to **control access to resources by assigning roles to users, groups, or service accounts**.

---

# 10. What is a Service Account in GCP?

A service account is an **identity used by applications or services to interact with GCP resources**.

---

# 11. What is Virtual Private Cloud (VPC) in GCP?

VPC is a **private network used to connect GCP resources securely**.

It provides:

- subnets
- firewall rules
- routing

---

# 12. What is GCP Load Balancing?

GCP Load Balancer distributes **incoming traffic across multiple instances to ensure high availability and performance**.

---

# 13. What is Cloud Monitoring?

Cloud Monitoring is used to **monitor performance, uptime, and health of GCP resources**.

---

# 14. What is Cloud Logging?

Cloud Logging collects and stores **logs from applications and GCP services**.

---

# 15. What is Infrastructure as Code in GCP?

Infrastructure as Code allows provisioning resources using code.

Common tools:

- Terraform
- Deployment Manager

---

# 16. What is Cloud Pub/Sub?

Pub/Sub is a **messaging service used for asynchronous communication between applications**.

---

# 17. What is Cloud Run?

Cloud Run is a **serverless platform to run containerized applications**.

It automatically scales based on traffic.

---

# 18. What is the difference between GKE and Compute Engine?

| Feature | GKE | Compute Engine |
|---|---|---|
Type | Managed Kubernetes | Virtual Machines |
Use Case | Container orchestration | General compute workloads |

---

# 19. What are common GCP use cases?

- web hosting
- containerized applications
- data analytics
- machine learning
- microservices

---

# 20. Why is GCP important for DevOps?

GCP helps DevOps teams:

- deploy scalable applications
- manage containers using Kubernetes
- automate infrastructure
- monitor systems effectively
