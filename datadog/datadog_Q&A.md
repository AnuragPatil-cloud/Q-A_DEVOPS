## 🔰 Basic Questions

### 1. What is Datadog?
Datadog is a **monitoring and analytics platform** used to monitor infrastructure, applications, logs, and metrics in real-time.

---

### 2. What are the main features of Datadog?
- Infrastructure Monitoring  
- Application Performance Monitoring (APM)  
- Log Management  
- Real-time dashboards  
- Alerting and notifications  
- Integration with cloud services (AWS, Azure, GCP)

---

### 3. What is a Datadog Agent?
Datadog Agent is a lightweight software installed on hosts to collect:
- Metrics  
- Logs  
- Events  

It sends this data to Datadog servers.

---

### 4. What are Metrics in Datadog?
Metrics are numerical data points that represent system performance over time.
Example:
- CPU usage  
- Memory usage  
- Disk I/O  

---

### 5. What are Logs in Datadog?
Logs are records of events generated by applications, servers, or systems.
They help in debugging and troubleshooting.

---

### 6. What is APM in Datadog?
APM (Application Performance Monitoring) helps track:
- Request flow  
- Latency  
- Errors  
- Service dependencies  

---

### 7. What is a Dashboard?
A dashboard is a visual representation of metrics using graphs and charts.

---

### 8. What is a Monitor?
Monitor is used to trigger alerts when a condition is met.
Example:
- CPU > 80%  

---

## ⚙️ Intermediate Questions

### 9. How do you install Datadog Agent on Linux?
```bash
DD_API_KEY=<your_api_key> bash -c "$(curl -L https://s3.amazonaws.com/dd-agent/scripts/install_script.sh)"
```

---

### 10. How does Datadog integrate with AWS?
- Uses IAM roles  
- Pulls metrics from CloudWatch  
- Auto-discovers services  

---

### 11. What is a Tag in Datadog?
Tags are key-value pairs used to filter and organize data.
Example:
```
env:production
region:us-east-1
```

---

### 12. Difference between Metrics and Logs?

| Metrics | Logs |
|--------|------|
| Numeric data | Text data |
| Lightweight | Detailed |
| Used for monitoring | Used for debugging |

---

### 13. What is Log Parsing?
Log parsing extracts meaningful fields from raw logs.

---

### 14. What are Alerts?
Alerts notify users when defined conditions are triggered.

---

### 15. What is a Trace in APM?
A trace represents the journey of a request across multiple services.

---

## 🔄 Scenario-Based Questions

### 16. CPU usage is high. How will you debug?
- Check metrics dashboard  
- Analyze CPU graphs  
- Check logs  
- Identify process causing issue  

---

### 17. Application is slow. What will you do?
- Use APM to check latency  
- Identify slow service  
- Analyze traces  
- Check DB or API delays  

---

### 18. Logs are not coming. What could be the issue?
- Agent not running  
- Wrong log path  
- Permission issue  
- Network issue  

---

### 19. How do you reduce alert noise?
- Set proper thresholds  
- Use anomaly detection  
- Use alert grouping  

---

### 20. How do you monitor Kubernetes using Datadog?
- Install Datadog Agent via Helm  
- Enable Kubernetes integration  
- Monitor pods, nodes, and containers  

---

## 🧠 Tips for Interview

- Understand **real-time use cases**
- Practice **basic commands and setup**
- Know **difference between monitoring tools**
- Be clear about:
  - Metrics vs Logs vs Traces  
  - Agent vs Integration  
- Explain answers with **practical examples**
